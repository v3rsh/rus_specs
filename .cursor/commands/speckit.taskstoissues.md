## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если он не пустой).

## План

1. Запустите `.specify/scripts/powershell/check-prerequisites.ps1 -Json -RequireTasks -IncludeTasks` из корня репозитория и распарсите список FEATURE_DIR и AVAILABLE_DOCS. Все пути должны быть абсолютными. Для одинарных кавычек в аргументах типа "I'm Groot" используйте синтаксис экранирования: например, 'I'\''m Groot' (или двойные кавычки, если возможно: "I'm Groot").
1. Из выполненного скрипта извлеките путь к **tasks**.
1. Получите Git remote, выполнив:

```bash
git config --get remote.origin.url
```

> [!CAUTION]
> ПРОДОЛЖАЙТЕ К СЛЕДУЮЩИМ ШАГАМ ТОЛЬКО ЕСЛИ REMOTE ЯВЛЯЕТСЯ GITHUB URL

1. Для каждой задачи в списке используйте GitHub MCP сервер для создания нового issue в репозитории, который соответствует Git remote.

> [!CAUTION]
> НИ ПРИ КАКИХ ОБСТОЯТЕЛЬСТВАХ НЕ СОЗДАВАЙТЕ ISSUES В РЕПОЗИТОРИЯХ, КОТОРЫЕ НЕ СООТВЕТСТВУЮТ REMOTE URL
